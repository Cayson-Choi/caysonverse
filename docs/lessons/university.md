# 대학생 — "AI 시대 개발 워크플로"

## 수업 목표
- **지식**: AI 페어코딩 시대의 개발 워크플로(설계 문서 → 프롬프트 → 구현 → 리뷰 → 테스트)를 실제 프로덕션 코드베이스로 이해한다
- **체험**: 실서비스 수준 코드베이스에서 AI와 함께 기능을 구현/리뷰해 본다
- **태도**: "AI가 코드를 짜 주는 시대에 개발자의 부가가치는 어디에 있는가"에 대한 자기 답을 만든다

## 대상 특성과 배려
- 전공/비전공 편차가 크다 → 코스를 2트랙으로: 비전공자는 직장인 커리큘럼(professionals.md)의 실습을, 전공자는 아래 코드베이스 실습을
- 이미 Copilot/ChatGPT로 과제하는 세대 → "AI 쓰는 법"이 아니라 **"AI로 프로덕션 품질을 내는 법"** 이 차별점
- 취업 불안이 실재 → 마무리 토론을 커리어 세션으로

## 이 코드베이스가 교재인 이유
caysonverse 자체가 수업 교재다. 다음이 모두 실물로 있다:
- **모노레포 구조**: npm workspaces (`shared`/`server`/`client`/`loadtest`) — 스키마·상수를 공유하는 실전 구조
- **실시간 멀티플레이 아키텍처**: Colyseus 서버 권위 상태, 10Hz 패치, 클라이언트 보간(150ms), 이동 검증(스피드핵/순간이동 차단)
- **설계 문서가 곧 프롬프트**: `docs/design.md`가 AI에게 준 설계 그 자체 — "문서를 잘 쓰는 것이 곧 코딩"의 실증
- **테스트·부하테스트**: 기능별 단위 테스트 + 100봇 부하 게이트(`loadtest/`, CPU<50%, RSS<400MB)
- **배포**: Railway config-as-code(`railway.toml`), 헬스체크, 재접속 복원력 설계

## 타임테이블 (90분 — 특강형)

| 시간 | 단계 | 내용 |
|---|---|---|
| 0–8분 | 접속 온보딩 | 접속하며 "지금 겪는 UX가 오늘 뜯어볼 코드"임을 예고 |
| 8–18분 | 미로 레이스 | 아이스브레이킹 + "이 미로 생성·탈출 판정도 오늘 볼 코드" |
| 18–48분 | 착석 강의 | ① 아키텍처 투어 (클라 R3F / 서버 Colyseus / shared 스키마) ② 제작 워크플로 공개 — 설계 문서 → AI 대화 → 리뷰 → 테스트의 실제 커밋 히스토리 ③ 서버 권위·보간 등 "AI가 짰지만 사람이 설계한" 부분 강조 |
| 48–75분 | **라이브 페어코딩** | 강사가 AI에게 실기능을 시키는 전 과정 공개: 요구사항 한 문장 → AI의 코드 탐색 → 구현 → 타입체크/테스트 → (가능하면) 커밋. 학생들이 요구사항을 투표로 결정 |
| 75–90분 | 토론 + 마무리 | "AI가 코드를 짜면 개발자는 뭘 하나" — 설계·검증·책임. Q&A |

**120분 변형 (실습형)**: 75분 이후를 조별 실습으로 — 레포를 클론해 각 조가 AI와 함께 작은 이슈 하나를 브랜치에서 구현, 마지막 15분에 조별 시연. (사전에 이슈 목록과 개발 환경 안내 필요)
**정규 커리큘럼(4주) 확장**: 1주 아키텍처 / 2주 AI 페어코딩 실습 / 3주 AI 코드리뷰·테스트 / 4주 기능 발표회 — 각 주차가 위 90분 구조의 변형.

## 진행 스크립트 요지
- 도입 훅: "이 멀티플레이 서버 코드, 몇 명이 짰을까? — 사람 0명. 하지만 설계 결정은 전부 사람이 했다. 그 경계선이 오늘 주제"
- 커밋 히스토리를 열어 보여주기: 리뷰 지적사항이 커밋 메시지에 남아 있는 것(예: "리뷰 v2-13 M2" 형태)으로 "AI 코드도 리뷰 사이클을 돈다"를 실증
- 부하테스트 게이트를 보여주며: "AI가 짠 코드를 믿는 방법은 신뢰가 아니라 검증"

# 중고등학생 — "AI 리터러시 + 바이브코딩 맛보기"

## 수업 목표
- **지식**: 생성형 AI의 원리(패턴 학습·확률적 생성)와 한계(환각·편향)를 설명할 수 있다
- **체험**: "말로 시켜서 소프트웨어가 만들어지는" 바이브코딩을 눈앞에서 목격하고, 직접 프롬프트로 결과물을 만들어 본다
- **태도**: AI를 베끼는 도구가 아니라 "내 능력을 증폭하는 도구"로 보는 관점 + AI 시대 진로 감각

## 대상 특성과 배려
- 이미 ChatGPT 등을 써 본 비율이 높다 → "AI 소개"는 건너뛰고 **원리와 한계**로 직행
- 게임 감각이 있다 → 조작 안내는 1~2분이면 충분, 미로 레이스는 진심으로 경쟁함
- 시니컬한 반응 대비 → 이론 강의를 줄이고 **라이브 데모**(실패 가능성 포함)로 승부
- 수행평가·과제에 AI를 쓰는 현실 → 윤리 토론을 설교가 아닌 찬반 토론으로

## 타임테이블 (90분)

| 시간 | 단계 | 내용 |
|---|---|---|
| 0–8분 | 접속 온보딩 | 접속, 캐릭터 선택, 조작 익히기 (빠르게) |
| 8–20분 | 미로 탈출 레이스 | 탈출 방송으로 실시간 순위. 상위권 호명 |
| 20–45분 | 착석 강의 | ① 이 세계의 제작기 — 설계 문서와 AI 대화만으로 만들어진 과정 ② 생성형 AI 원리 5분 요약 ③ 환각·편향 사례 ④ "그래서 너희 진로에 무슨 의미인가" |
| 45–65분 | **라이브 바이브코딩 데모** | 강사가 그 자리에서 AI에게 기능 하나를 시킨다 (아래 데모 후보). 코드가 바뀌고 → 새로고침하면 → 모두의 화면에서 세계가 바뀐다 |
| 65–83분 | 실습 + 토론 | (A) 프롬프트 실습: AI 도구로 미니 결과물(웹페이지/이야기/이미지) 만들기 경쟁 (B) 채팅 찬반 토론: "AI로 숙제하는 것, 어디까지 괜찮은가" |
| 83–90분 | 마무리 | 결과물 공유, 갤러리 관람(선택), 진로 한마디 |

**60분 변형**: 미로 8분, 강의 18분, 라이브 데모 15분, 실습·토론은 택1로 12분.
**120분 변형**: 실습을 조별 프로젝트(각 조가 만들 기능을 프롬프트로 기획 → 강사가 대표 1~2개를 라이브로 실행)로 확장.

## 라이브 데모 후보 (리스크 낮은 순)
1. 채팅 이모지 종류 1개 추가 / 말풍선 표시 시간 변경 — 실패 확률 낮음, 변화가 눈에 보임
2. 새 조작키 추가 (예: 실제 사례 — "클릭 이동 중 Esc를 누르면 멈추게 해줘" 한 문장으로 구현된 기능)
3. 강의실 스크린 문구를 그 학교/반 이름으로 교체
- 데모 원칙: **학생들이 그 자리에서 시킬 기능을 투표로 고르게 하면 몰입이 2배가 된다.**
  실패해도 "AI와의 개발은 이렇게 반복하는 것"이라는 교육 소재로 쓴다.
- 사전에 dev 서버 분리 실행 (수업 접속 중인 운영 서버에 배포하지 않는다 — 전원 퇴장됨)

## 진행 스크립트 요지
- 도입 훅: "이 세계는 코딩 학원 안 다닌 사람이 AI랑 대화해서 만들었다. 오늘 그 과정을 눈으로 보여준다"
- 원리 설명은 채팅을 활용: "AI처럼 대답해 보자 — '옛날 옛적에' 다음에 올 말을 채팅으로" → 확률적 다음 단어 예측을 몸으로 이해
- 진로 메시지: "AI가 대체하는 것은 직업이 아니라 작업. 시키는 능력(문제 정의력)이 새 실력"

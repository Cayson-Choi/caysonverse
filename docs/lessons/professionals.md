# 직장인 — "개발자가 아니어도 이걸 만든다"

## 수업 목표
- **지식**: 생성형 AI로 아이디어가 실제 서비스가 되기까지의 과정·기간·비용 감각을 얻는다
- **체험**: 각자 업무의 반복 작업 하나를 AI로 프로토타이핑해 본다
- **태도**: "우리 팀/우리 회사에서 뭘 자동화할 수 있나"를 스스로 목록화하는 관점 전환

## 대상 특성과 배려
- 시간이 곧 비용 → 이론 최소화, **ROI와 실사례** 중심. 미로 레이스도 "게임"이 아니라 "온보딩 UX 관찰"로 프레이밍하면 몰입이 좋다
- 직무 다양성(기획·마케팅·총무·엔지니어) → 실습은 직무 불문 공통 포맷("내 업무의 반복 작업 1개")으로
- 회사 보안 정책 민감 → 사내 데이터 반입 금지 원칙과 대안(가상의 예시 데이터)을 명시

## 이 사이트가 최고의 사례인 이유
- **기획→배포 전 과정이 투명**: 설계 문서(docs/design.md), 커밋 히스토리, Railway 배포 설정까지 전부 공개 가능
- **비용 구조가 충격적**: 에셋 전부 무료(CC0), 서버 1대(1vCPU/1GB)로 100명 동시 접속, TTS는 브라우저 내장(비용 0)
- **"사실상 1인 제작"**: 외주 견적으로 수천만 원대일 3D 멀티플레이 서비스를 AI와의 대화로 제작 — "여러분 업무 도구는 이것보다 훨씬 쉽다"는 설득 구조

## 타임테이블 (90분)

| 시간 | 단계 | 내용 |
|---|---|---|
| 0–8분 | 접속 온보딩 | 접속 체험 자체가 첫 교보재: "가입 없는 온보딩이 이탈률을 어떻게 바꾸는가" |
| 8–18분 | 미로 레이스 | 아이스브레이킹 (팀 대항으로 하면 부서 간 분위기가 풀림) |
| 18–45분 | 착석 강의 | ① 이 서비스의 제작기: 기간·비용·과정 공개 ② "AI 도입"의 실체 3단계 — 개인 생산성 → 팀 워크플로 → 제품 ③ 실패 사례와 한계(환각, 보안, 검증 비용)도 정직하게 |
| 45–55분 | 라이브 데모 | 그 자리에서 AI에게 기능 하나를 시켜 5분 만에 반영되는 것을 시연 ("Esc로 이동 멈추기"가 실제 한 문장으로 구현된 사례) |
| 55–80분 | 실습 워크숍 | 각자 "내 업무에서 매주 반복하는 작업 1개"를 골라 AI 도구로 프로토타입 (메일 초안 자동화, 회의록 요약, 보고서 표 정리 등). 결과를 채팅으로 공유 |
| 80–90분 | ROI 토론 + 마무리 | "이걸 우리 팀에 도입하면 주당 몇 시간?" 어림 계산 공유, 도입 시 장애물(보안·결재) 논의 |

**60분 변형(점심 특강)**: 미로 생략, 강의 25분 + 데모 10분 + 실습 15분.
**120분 변형(워크숍)**: 실습을 조별로 확장 — 조마다 자동화 대상을 하나 정해 프로토타입 + 3분 발표.

## 진행 스크립트 요지
- 도입 훅: "지금 접속하신 이 3D 서비스, 외주 견적을 받으면 얼마짜리일까요? 제작비를 공개합니다"
- ROI 프레임: 도구 비용(월 구독료) vs 절약 시간 × 시급 — 계산식을 그대로 따라 하게
- 보안 메시지: "사내 데이터를 넣지 말고, 데이터의 '모양'만 가짜로 만들어 넣어라" — 실습 규칙으로 강제
- 마무리 메시지: "오늘 만든 프로토타입을 내일 팀 회의에 가져가는 것"이 이 수업의 성공 지표

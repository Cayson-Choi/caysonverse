# 수업용 기능 개발 로드맵

수업 기획안 5종이 실제로 요구하는 기능만 역산해 우선순위를 매긴 로드맵입니다.
현재 사이트만으로도 모든 수업이 진행 가능하며(기획안이 그렇게 설계됨),
아래 기능은 수업 품질을 올리는 순서대로 추가합니다.

## P1 — 강의실 스크린 콘텐츠 교체 (모든 수업이 사용)

**현재**: 강의실 대형 스크린에 환영 문구가 고정 표시.
**목표**: 관리자가 수업 중 스크린의 문구(나아가 이미지)를 실시간으로 교체.

- 사용처: 모든 대상 — 강의 단계에서 슬라이드 핵심 문구 표시, 학교/기관명 환영 문구
  (중고등 라이브 데모 후보이기도 함 — 수업 중 이 기능 자체를 만들어 보이는 극적 효과)
- 구현 스케치: 기존 관리자 공지(announce) 경로와 동일한 패턴 — 관리자 패널에 입력 UI 추가,
  서버 스키마에 스크린 상태 1필드 추가, 늦게 입장해도 보이도록 상태(schema)로 동기화
- 규모: 소 (기존 공지 배너 인프라 재사용)

## P2 — 퀴즈 존 (초등·중고등·시니어 활동의 핵심)

**목표**: 바닥에 O/X (또는 1~4번) 구역을 표시하고, 참가자가 답 구역으로 걸어가 서면
관리자 화면에 구역별 인원이 집계되는 골든벨.

- 사용처: 초등(AI 상식 OX), 중고등(환각·윤리 딜레마 투표), 시니어(피싱 구별 OX)
- 이 사이트의 강점과 정합: "몸(아바타)으로 답한다"는 3D 공간만의 상호작용 —
  채팅 골든벨은 어디서나 되지만 이건 메타버스에서만 됨
- 구현 스케치: 라운지 바닥에 구역 데칼 + 서버에서 플레이어 좌표로 구역 판정(이미
  위치는 서버 권위) + 관리자에게 집계 브로드캐스트. 문제 출제는 P1 스크린 재사용
- 규모: 중

## P3 — AI NPC 도슨트 (체험을 사이트 안에서 완결)

**목표**: 갤러리/라운지에 AI 챗봇 NPC를 세우고, 가까이 가서 채팅하면 AI가 응답.

- 사용처: 초등("AI랑 대화" 체험을 별도 도구 없이), 시니어(강사 대리 입력 없이 직접 대화),
  전 대상(수업 전 대기 시간의 볼거리)
- 현재 기획안은 외부 AI 도구로 이 체험을 대체 중 — P3가 생기면 기기 전환 없이 완결됨
- 구현 스케치: NPC 아바타 1체 + 근접 시 대화 UI, 서버에서 Claude API 프록시
  (키는 서버 환경변수, 클라이언트 노출 금지), 남용 방지 레이트리밋(기존 채팅 3회/5초
  패턴 재사용), 시스템 프롬프트로 "갤러리 도슨트" 페르소나 고정
- 규모: 중~대 (API 비용 발생 — 수업 시간에만 활성화하는 토글 필요)

## P4 — 보류 확인: 화면 공유 / 음성

- 설계는 docs/design.md 15장(LiveKit)에 이미 보존되어 있음
- 현재 기획안은 전부 "빔프로젝터(오프라인) / Zoom 병행(온라인)"으로 우회하므로 급하지 않음
- 착수 조건: 온라인 원격 수업 비중이 커지고, 사이트 하나로 완결해야 할 때

## 로드맵 외 — 운영에서 이미 확인된 소소한 개선 후보

- 수업용 조작 도움말 오버레이(? 버튼) — 시니어·초등 온보딩 시간 단축
- 관리자용 "전원 소환"(라운지로 모으기) — 단계 전환 시간 단축 (현재는 공지+구두 안내)

이 항목들은 실제 수업을 1회 이상 운영해 본 뒤 병목이 확인되면 우선순위에 편입한다.

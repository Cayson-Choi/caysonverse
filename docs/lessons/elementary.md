# 초등학생 — "AI랑 친구 되기"

## 수업 목표
- **지식**: AI는 많은 예시를 보고 배우는 프로그램이라는 것을 자기 말로 설명할 수 있다
- **체험**: AI가 만든 3D 세계에서 놀아보고, AI에게 그림/이야기를 부탁해 본다
- **태도**: AI는 무섭거나 마법 같은 게 아니라 "잘 부탁하면 잘 도와주는 도구"임을 느낀다

## 대상 특성과 배려
- 집중 시간이 짧다 → 한 활동 10~15분 단위로 끊고, 몸(아바타)을 움직이는 활동을 교차
- 태블릿 사용 비율이 높다 → 조이스틱·터치 조작 지원이 이미 있으므로 기기 제약 없음.
  단, 첫 조작 연습 시간을 다른 대상보다 길게(5분 → 8분) 잡는다
- 타자가 느리다 → 채팅 대신 **이모지 6종(숫자키/버튼)** 을 주 리액션 도구로
- 닉네임에 실명 대신 별명을 쓰게 한다 (개인정보 습관 교육을 겸함)

## 타임테이블 (90분)

| 시간 | 단계 | 내용 |
|---|---|---|
| 0–12분 | 접속 온보딩 | QR 접속, 별명·캐릭터 고르기, 걷기/클릭이동/이모지 연습 |
| 12–27분 | 미로 탈출 레이스 | "AI가 만든 미로에서 탈출하라!" 탈출 방송으로 순위 발표 |
| 27–50분 | 착석 강의 | 강의실 착석. ① 방금 논 이 세계를 누가 만들었을까? → "AI에게 말로 시켜서 만들었어요" ② AI는 어떻게 배울까 (강아지 사진 10,000장 비유) ③ 갤러리 예고: "AI가 그린 그림도 있어요" |
| 50–75분 | AI 체험 활동 | 조별로 AI 이미지 생성 도구 체험: "우리 반 상상 동물" 그리기 — 부탁(프롬프트)을 자세히 할수록 잘 나온다는 것을 발견하게 유도 |
| 75–90분 | 갤러리 관람 + 마무리 | 갤러리방(AI가 그린 1살~100살 초상) 관람 → 라운지에서 소감을 이모지로 표현, 퇴장 인사 |

**60분 변형**: 미로 10분, 강의 18분, 체험 15분으로 압축. 갤러리는 강의 중 언급만.
**120분 변형**: 체험을 2회전(그림 → 이야기 만들기)으로 확장.

## 진행 스크립트 요지
- 강의 도입은 반드시 질문으로: "여러분이 방금 뛰어다닌 이 세계, 사람 몇 명이 몇 달 걸려 만들었을까요?" → "선생님이 AI랑 **대화해서** 만들었어요"가 이 수업의 한 방 문장
- "AI에게 부탁 잘하기" 개념을 게임처럼: "그림 그려줘" vs "무지개색 날개가 달린 아기 고양이 그려줘" 결과를 비교해서 보여주기 → 프롬프트라는 단어는 마지막에만 소개
- 미로 레이스는 탈출 방송(🎉 ○○님이 탈출!)이 자동으로 뜨므로 강사는 중계만 하면 됨
